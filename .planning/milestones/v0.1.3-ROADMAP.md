## Milestone Summary

**Key Decisions:**

- **Single source of truth = package.json** — `runtime-deps.json` removed entirely. `--omit=dev` flag already provides runtime-only behavior. No more drift between two manifests.
- **Validate, don't guess** — install-deps.sh and mcp-wrapper.sh's file-existence checks replaced with `require("better-sqlite3")` validation. Fixes the Node 25 binding bug class permanently.
- **Migrations grouped with runtime code** — each REQ atomic to one phase. Rejected the alternative ("all migrations first, then wiring") because it leaves migration tests unverified against the code that exercises them.
- **Brought `/arcanon:upload` removal forward to v0.1.3** — originally planned for v0.2.0. v0.1.2 already shipped a breaking change (LIGAMEN_* purge); one more removal in the same wave is consistent.
- **`{xxx}`-only path canonicalization, no backfill** — Express `:id` style deferred. No backfill of historic rows because re-canonicalizing would silently collapse legitimately-distinct connections.
- **Whole-file evidence substring (not ±3 lines)** — agent schema has no `line_start` column; faithful interpretation of REQUIREMENTS-stated `±3 lines` documented as Phase 109 D-03. Future ticket can narrow once `connections.line_start` lands.
- **`/arcanon:verify` is read-only** — purely reports per-connection verdicts; doesn't update or delete. Action commands (`/arcanon:correct`, `/arcanon:rescan`) deferred to v0.1.5.

**Issues Resolved Beyond REQs:**

1. **Migration 013 numbering collision** — Phases 109 and 110 both initially claimed migration 013. Caught at planning review, renumbered to 013/014/015/016 via commit `b39ff28` BEFORE execution. Zero impact.
2. **`UNIQUE INDEX uq_connections_dedup` missing on connections table** — Phase 109-02 executor noticed `INSERT OR REPLACE` couldn't actually collapse rows without a UNIQUE constraint. Added the index to migration 013.
3. **`upsertService` `lastInsertRowid` poisoning** — sibling INSERTs were polluting the connection-level rowid, breaking `serviceIdMap` on idempotent re-scans. Phase 109-02 fixed via explicit `SELECT id ...` lookup post-upsert.
4. **MCP test hang issue** — Phase 111-03 caught project-wide MCP server stdio transport hanging when imported from a test. Gated behind `NODE_TEST_CONTEXT` env var.
5. **v0.1.2 documented test failure resolved** — `worker/mcp/server-search.test.js queryScan` pre-existing failure (carried since v0.1.2 ship) now passes after Phase 107-112 work.

**Issues Deferred (filed):**

- 1 pre-existing node test failure: `worker/scan/manager.test.js` incremental-prompt mock missing `_db`. Carried from v0.1.2.
- macOS HOK-06 p99 latency caveat (documented since v0.1.1) — platform constraint, not a code bug. CI continues to use threshold=100.
- `commands/update.md:21` `claude plugin update --help` reference — upstream Claude Code host CLI flag, not an Arcanon flag. Permanent VER-04 exception. v0.1.4's `--help` system can refine grep to `/arcanon:.*--help`.
