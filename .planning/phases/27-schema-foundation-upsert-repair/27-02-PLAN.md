<objective>
Add a naming convention rule to the agent prompt that instructs the scan agent to derive service names from the package manifest name field (package.json, pyproject.toml, go.mod) and format them as lowercase-hyphenated. Block generic names (server, worker, api, app, main) as explicitly disallowed.

Purpose: _resolveServiceId() does an unscoped name lookup — if two repos each have a service named "api" or "worker", their connections merge into phantom edges. Enforcing consistent manifest-derived names at the agent output level prevents this identity collision from growing. This is the cheapest fix: no schema change, no code change — just a prompt update.

Output: worker/scan/agent-prompt-deep.md (modified — new "Service Naming Convention" section added before the Output Format section).

  <action>
Add a new section "## Service Naming Convention" to worker/scan/agent-prompt-deep.md, inserted between the "## What NOT to Report" section and the "## Output Format" section.

The section content must convey these rules:

1. Derive the service name from the project's package manifest name field:
   - Node.js/npm: use the "name" field from package.json
   - Python: use the "name" field from pyproject.toml [project] section, or the top-level package name from setup.py
   - Go: use the module name from go.mod (last path segment)
   - Rust: use the "name" field from Cargo.toml [package] section
   - If no manifest is found, use the repository directory basename

2. Format the name as lowercase-hyphenated:
   - Convert all characters to lowercase
   - Replace underscores and spaces with hyphens
   - Remove any leading @scope/ prefixes (e.g., "@company/auth-service" becomes "auth-service")
   - Example: "AuthService" -> "auth-service", "user_api" -> "user-api", "@acme/payment-gateway" -> "payment-gateway"

3. Do NOT use generic names. The following names are DISALLOWED because they cause false identity merges across repos: server, worker, api, app, main, service, backend, frontend. If the manifest name field is one of these, append a disambiguating suffix derived from the directory or module path (e.g., "api" -> "auth-api" if the service is in an auth directory).

4. The service name in the JSON output must be stable across scans — use the manifest name, not a runtime hostname or container tag.

Also update the "name" field description in the JSON Output Schema section (line 138 area):
  Change: "name": "string — service identifier"
  To: "name": "string — service identifier, MUST be lowercase-hyphenated, derived from package manifest name field (package.json 'name', pyproject.toml [project] name, go.mod module last segment, Cargo.toml [package] name). See Service Naming Convention section."

Do NOT change any other sections of the prompt. Preserve all existing content exactly.
