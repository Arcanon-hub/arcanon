Use db.exec() for all DDL and bulk DML. Never use db.prepare().run() for CREATE TABLE or multi-row DML.

Implementation steps inside up(db):

Step 1 — Rename old table:
  ALTER TABLE services RENAME TO services_old;

Step 2 — Create new table with constraint:
  CREATE TABLE services (
    id             INTEGER PRIMARY KEY AUTOINCREMENT,
    repo_id        INTEGER NOT NULL REFERENCES repos(id),
    name           TEXT    NOT NULL,
    root_path      TEXT    NOT NULL,
    language       TEXT    NOT NULL,
    type           TEXT    NOT NULL DEFAULT 'service',
    canonical_name TEXT,
    UNIQUE(repo_id, name)
  );

Step 3 — Build id remapping table (maps every old id to the surviving MAX(id) for its pair):
  CREATE TEMP TABLE _svc_id_map AS
  SELECT s.id AS old_id,
         (SELECT MAX(s2.id) FROM services_old s2
          WHERE s2.repo_id = s.repo_id AND s2.name = s.name) AS new_id
  FROM services_old s;

Step 4 — Copy deduplicated rows:
  INSERT INTO services (id, repo_id, name, root_path, language, type)
  SELECT MAX(id), repo_id, name, root_path, language, type
  FROM services_old
  GROUP BY repo_id, name;

Step 5 — Re-point connections (source):
  UPDATE connections
  SET source_service_id = (SELECT new_id FROM _svc_id_map WHERE old_id = source_service_id)
  WHERE source_service_id IN (SELECT old_id FROM _svc_id_map WHERE old_id != new_id);

Step 5b — Re-point connections (target):
  UPDATE connections
  SET target_service_id = (SELECT new_id FROM _svc_id_map WHERE old_id = target_service_id)
  WHERE target_service_id IN (SELECT old_id FROM _svc_id_map WHERE old_id != new_id);

Step 6 — Re-point exposed_endpoints:
  UPDATE exposed_endpoints
  SET service_id = (SELECT new_id FROM _svc_id_map WHERE old_id = service_id)
  WHERE service_id IN (SELECT old_id FROM _svc_id_map WHERE old_id != new_id);

Step 7 — Drop old tables:
  DROP TABLE services_old;
  DROP TABLE _svc_id_map;

Step 8 — Rebuild all FTS5 indexes (must be last step, inside same transaction):
  INSERT INTO services_fts(services_fts) VALUES('rebuild');
  INSERT INTO connections_fts(connections_fts) VALUES('rebuild');
  INSERT INTO fields_fts(fields_fts) VALUES('rebuild');

All 8 steps should be in a single db.exec() call or in separate db.exec() calls — both are fine because the transaction wrapper is external.

Make two targeted changes to worker/db/query-engine.js:

CHANGE 1 — Replace _stmtUpsertService (constructor, around line 264).

Remove:
  this._stmtUpsertService = db.prepare(
    "INSERT OR REPLACE INTO services (repo_id, name, root_path, language, type) VALUES (@repo_id, @name, @root_path, @language, @type)"
  );

Replace with:
  this._stmtUpsertService = db.prepare(
    "INSERT INTO services (repo_id, name, root_path, language, type) VALUES (@repo_id, @name, @root_path, @language, @type) ON CONFLICT(repo_id, name) DO UPDATE SET root_path = excluded.root_path, language = excluded.language, type = excluded.type"
  );

CRITICAL: Use excluded.root_path / excluded.language / excluded.type (the excluded. qualifier references the incoming proposed values, not the existing row values). Without excluded., the SET would be a no-op.

IMPORTANT: After ON CONFLICT DO UPDATE, better-sqlite3's result.lastInsertRowid returns the id of the existing row that was updated (not a new id). This means upsertService() already correctly returns the preserved id — no change needed to the upsertService() method body.

CHANGE 2 — Remove the MAX(id) GROUP BY workaround from getGraph() (around line 543-552).

Remove this WHERE clause from the services SELECT in getGraph():
  WHERE s.id IN (SELECT MAX(id) FROM services GROUP BY name)

The updated services query in getGraph() should be simply:
  SELECT s.id, s.name, s.root_path, s.language, s.type, s.repo_id, r.name as repo_name, r.path as repo_path
  FROM services s
  JOIN repos r ON r.id = s.repo_id

No other changes. Do not touch any other prepared statements, methods, or logic.

NOTE: The connections query in getGraph() already does not filter by service id — it joins directly. Leave it unchanged.
